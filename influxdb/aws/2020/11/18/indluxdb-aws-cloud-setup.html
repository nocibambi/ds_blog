Setting up AWS Cloud for InfluxDB

How to set up AWS Cloud for InfluxDB?

Nov 18, 2020 • 1 min read

InfluxDB   AWS

Here are the preparation steps you need to do if you would like to use AWS Cloudwatcher with InfluxDB.

Based on this post

  1. Create an AWS account
  2. Select the appropriate region:
  3. Search for “EC2” in the Find Services search box and select EC2 (Virtual Servers in the Cloud) (Please not that you have to provide credit card information and it may take up to 24 hours for amazon to activate):
  4. Select Key Pairs and the create a key pair:
  5. Download the keys.
  6. Make the key readable only by the owner: chmod 400 name-of-the-key.pem
  7. Open IAM from the Services menu:
  8. Create a user and grant permissions to it:
    1. Select Add user:
    2. Name the user and add access rights:
    3. On the Set Permissions screen click on Add user to group and Create a group:
    4. Define a name and add a policy for the new group:
    5. At the end of the process you will see your access details:

    Note that AWS does not save your secret after you close your final screen. On the other hand, you will always be able to create a new key for your users.

  9. Install the AWS CLI on your system.

    On a linux machine, you can do this as here:

     curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
 unzip awscliv2.zip
 sudo ./aws/install

    Verify your version:

     aws --version

    For further instructions see here.

  10. Also create a folder where you will store your keys and your yaml configuration.

     mkdir ~/Projects/influxdb_aws_cloudformation
 cd ~/Projects/influxdb_aws_cloudformation
 mv ~/Downloads/name-of-the-key.pem .
 touch config.yml
  11. Configure AWS with aws configure. This is where you need to add your Access Key ID, your Secret Access Key, and the region name (e.g. eu-central-1). You can skip the output format.
